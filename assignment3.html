WDD3 Assignment

You will be assessed on the basis of this assignment. Your work will be presented on paper for assessment.

The assignment will involve the production of a detailed functional specification document for your own individually selected project (WDD4). This should include the aspects of the course covered in WDD3 that apply to your project. These aspects include:

  • Sites goals and business analysis
  • User analysis (may include personas)
  • Task analysis (may include scenarios)
  • Technical requirements
  • Site map / information architecture
  • Workflows (interaction design)
  • Prototypes (mockups)
  • Costings
  • Development timescales

You are not necessarily expected to cover everything on this list, only what you judge to be appropriate for communicating the details regarding the structure and function of your site. You will, however, need to demonstrate that you have a clear understanding of user-centred design, information architecture and Web usability.

The documentation should draw on class work, previous assignments, and other recommended sources. The end products you put forward for assessment should already be fully reviewed by you (and others if appropriate) for accuracy, consistency and completeness. They should therefore be in a form that presents a professional approach - well presented and lacking in ambiguity.

This assignment should be between 2000 and 3000 words.
