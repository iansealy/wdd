Workaround Problems

Problems with pages that use workarounds:

  • Pages are larger (especially if images are used) and take longer to download
  • Pages are harder to maintain because:
    • the content is more complex
    • the content is harder to read
    • changing the style of a set of pages involves altering every page and often each individual tag (e.g. if want to change <font color="green"> to <font color="red">)

Both of these mean users waste time and so do authors. Since time is money, it would be much better to separate the content and the styling. Authors can concentrate on the content and designers can easily change the design.

Cascading Stylesheets

Cascading Stylesheets (CSS or just stylesheets) allow content to be separated from formatting

Examples of sites that use CSS rather than tables for page formatting:

  • http://www.adaptivepath.com/
  • http://www.joelsundkvist.com/
  • http://www.csszengarden.com/
  • http://www.strangebanana.com/

Simple Example

<style type="text/css">
body { color : red; }
</style>

CSS is the bit between the <style> tags

<style> tag must go in head of page (like the <title> tag).

All content (everything within the body tags) is red.

Syntax

body { color : red; }

Part before the curly brackets (in this case the body tag) is the selector. Part within the curly brackets is the declaration.

Each declaration consists of a property name (in this case color) and its value (in this case red). The name and value must be separated with a colon.

Can have more than one selector, for example:
h1, h2 { color : red; }

More Syntax

Can have more than one property, for example:
body { color : red; background-color : green;}

Properties must be separated with semicolons.

A stylesheet is just a collection of these rules.

Location of Styles

  • Internal stylesheet
    <style> tag appears in head of document. Can use comments (<!-- and -->) to prevent older browsers displaying styles.
  • External stylesheet
    A .css file contains the styles. Use this with:
    <link rel="stylesheet" type="text/css" href="style.css">
    Goes in the head of document.
    File just contains CSS (no HTML, so no <style> tag)
  • Inline styles
    Within a single HTML tag
    <p style="color : blue">

Classes

Styles can be applied to any HTML tags, but what if you want some h1 tags to have one style, but other h1 tags to have another style?

Use classes. Can define different classes of style. For example:

h1.warning { color : red }
h1.ok { color : green }

Apply specific class to h1 tags:

<h1 class="warning">Red / Warning Heading</h1>
<h1 class="ok">Green / OK Heading</h1>

More Classes

Don't have to associate classes with a specific tag, for example:

h1.yuk { color : red; background-color : green; }

<h1 class="yuk">Horrid style</h1>
<p class="yuk">Normal style</p>

.yuk { color : red; background-color : green; }

<h1 class="yuk">Horrid style</h1>
<p class="yuk">Horrid style</p>

Comments

Can add comments to stylesheets using /* and */. For example:

h1 { color : red; } /* Make headings red */
p { color : blue; } /* Make paragraphs blue */

Stylesheet Precedence

What if an inline style says that paragraphs should be blue, but an external stylesheet says they should be green?

Styles have an order of precedence:

  • Inline styles
  • Embedded styles
  • Linked styles
  • Default browser styles

So an inline style overrides an external style.

Font Properties

  • Named fonts
    p { font-family : Verdana, helvetica, sans-serif; } /* Go from specific to generic font */
  • Font size
    p { font-size : 16pt; }
    Can define size using points (e.g. 16pt), percent (e.g. 150% or 1.5 times larger than the current default), ems (e.g. 1.5em or 1.5 times larger than the current default), pixels (e.g. 30px), etc...
    Recommend percent or ems
  • Font style
    p { font-style : italic; }

More Font Properties

  • Font weight
    p { font-weight : bold; }
    p { font-weight : normal; }
  • Text transform
    p { text-transform : uppercase; }
    p { text-transform : lowercase; }
    p { text-transform : capitalize; }
  • Text decoration
    p { text-decoration : underline; }
    p { text-decoration : overline; }
    p { text-decoration : none; }
    p { text-decoration : line-through; }
    p { text-decoration : blink; }

Margin Properties

p { margin-left : 50px; }
p { margin-right : 20%; }
p { margin-top : 50px; }
p { margin-bottom : 3cm; }
p { margin : top right bottom left; }

CSS Support

New Web browsers (IE6, Netscape 7+, Opera 7+, Mozilla, Firefox) have good CSS support, but older browsers don't.

Sites that use CSS should always be thoroughly tested on a range of browsers (i.e. different types of browser and different versions of each type of browser).
