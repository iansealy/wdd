RSS News feeds

RSS can stand for a variety of things:

  • RDF Site Summary
  • Really Simple Syndication
  • Remote Site Summary
  • Rich Site Summary

Assume it doesn't stand for anything and just use the acronym.

RSS News feeds

RSS is an XML format designed for sharing news headlines or "What's new?" for a particular Web site.

  • The feed is updated as and when something new appears on your Web site
  • People subscribe to the feed using:
    • Feed readers e.g. Feedreader or Feedly
    • Email client e.g. Thunderbird
    • Phones
  • Also, other Webmasters pick up the feed and present a list of your current headlines on their site. These headlines link back to the original articles on your site.

Creating an RSS feed?

Code example

So, how do you create your own RSS feeds?

  • By hand - time consuming and you're liable to introduce errors.
  • Using a program on your server that queries a database and produces the RSS for you - requires technical knowledge and ability to run scripts and databases on your server.
  • The feed might be already provided if you use a third-party blogger/news group or even have your own blog software (such as WordPress) installed.

Finding feeds

Where should you look for feeds:

  • On the Web site itself e.g. AES
  • In the browser
  • From a searchable directory

Using feeds

Incorporating feeds:

  • Often done by a custom script on the server
  • There are JavaScript alternatives (such as RSSinclude) that do the processing for you and can just be pasted into your Web page

Podcasting

What is podcasting

Podcasting is a method of publishing audio broadcasts (usually mp3 files) via the Web. It allows users to subscribe so they automatically receive new broadcasts as they are published.

Called podcasts as they were originally intended for use on iPods. However, can be downloaded and listened to on a variety of devices.

Similar to RSS but uses <enclosure> tag.

SVG

What is SVG?

Scalable Vector Graphics

  • It is a vector-based graphics format
  • XML application
  • Open format
  • It's "readable" - view source
  • Huge accessibility advantages
  • Can be animated
  • Uses CSS

What is SVG? continued

  • You can add metadata to an image
  • Small filesize
  • W3C recommendation
  • Interoperability
  • Most modern browsers support SVG natively. Microsoft only introduced support in IE9.
  • Can also use SVGweb which provides support for older browsers using JavaScript and Flash.

SVG animation

Example - animation supported modern versions of Firefox, Opera and Chrome.


<?xml version="1.0" standalone="no"?>
<!DOCTYPE svg PUBLIC "-//W3C//DTD SVG 20001102//EN"
"http://www.w3.org/TR/2000/CR-SVG-20001102/DTD/svg-20001102.dtd">
<svg width="200" height="200" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink">

	<text id="name" font-size="100%" x="24" y="118">KIEREN
	<animate attributeName="font-size" attributeType="XML" begin="0s"

	dur="7s" fill="freeze" from="0" to="40" />
	</text>

	<g>
	<circle	cx="100" cy="100" r="75" fill-opacity="1.0"
	stroke-width="6" style="fill:blue;stroke:green;">

	    <animate attributeName="cx" attributeType="XML"
	             begin="0s" dur="9s" fill="freeze" from="0" to="100" />

	    <animate attributeName="cy" attributeType="XML"
	             begin="0s" dur="9s" fill="freeze" from="0" to="100" />

	    <animate attributeName="r" attributeType="XML"
	             begin="0s" dur="7s" fill="freeze" from="0" to="75" />

	    <animate attributeName="fill-opacity" attributeType="XML"
	             begin="9s" dur="7s" fill="freeze" from="1" to="0.1" />

	    <animate attributeName="stroke-width" attributeType="XML"
	             begin="9s" dur="7s" fill="freeze" from="6" to="1" />

	</circle
	</g>
</svg>

SVG supports:

Predefined shapes:

  • Rectangles
  • Circles
  • Ellipses
  • Polygons
  • Lines
  • Polylines
  • Paths/Bezier curves

SVG also supports:

  • Fill
  • Stroke
  • Text
  • Gradients
  • Masks
  • Opacity and visibility
  • Transformations
    • Rotation
    • Scale
    • Skew
    • Movement (translation)
  • Layers - position an SVG over a photograph

This means that complex images are possible

SVG authoring

  • By hand (can view source)
  • Adobe Illustrator
  • Corel Draw
  • Inkscape

Importantly, due to the open format, it is easy for CGI scripts (or developers) to produce SVG code "on the fly".

To include in a web page use the <svg> element just like <img>.

Agent Adoption

SVG is becoming increasingly popular (used throughout Wikipedia).

  • Already integrated into most desktop browsers and on iPads
  • Backwards compatible (text within the SVG will supply the location of a raster/bitmap images if SVG is unsupported)
  • Some mobile phones
  • Adobe Acrobat (PDFs)

HTML5 canvas element

  • <canvas>
    • An empty container that you can use for drawing or animation using JavaScript.
  • Support for:
    • Common shapes (circles, rectangles etc.)
    • Arcs and curves
    • Fills, strokes and shadows
    • Gradients
    • Transparency
    • Transformations (scaling, rotation, translation)
    • Text
    • Import other images or data

<canvas> element

<canvas id="mypicture" width="150" height="150"></canvas>

The canvas element can only be drawn in using JavaScript

<script type="text/javascript">
var c=document.getElementById("mypicture");
var cxt=c.getContext("2d");
cxt.fillStyle="#0D8DAD";
cxt.beginPath();
cxt.arc(75,75,30,0,Math.PI*2,true);
cxt.closePath();
cxt.fill();
</script>
