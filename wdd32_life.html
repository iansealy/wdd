The need for a process: the web development lifecycle

How do we create a great user experience?

  • First, we need a well-planned, methodical process (creativity needs to be harnessed!)
  • You can't just make it up as you go along

See Andy Budd's article on the importance of process.

The benefits of planning

  • No such thing as a perfect/foolproof process
  • Select / modify process as appropriate (goal, success factors, team-size, skillsets, technology, safety critical? etc.)
  • Quality, Time, Cost Triangle
  • Cost of change: $1 (planning) - $10 (during build) - $100 (post-launch) (Average software project spends 80% of its time on unplanned reworking!)
  • Poor planning >> a lot of time correcting mistakes. Understanding project progression means you are in control
  • All approaches can/should be user-centred

User-centred design

  • Web sites are built for end-users, so end-users should be the focus of our design and development
  • A user-centred design (UCD) approach is very powerful
  • There are many possible UCD models, but all follow the same sort of pattern

There is also an International Standard for user-centred design, ISO 13407, which can be used to inform many Web development projects.

The importance of documentation

Documentation is essential (even though people hate doing it) because:

  • Shared vision and Communication channel for a project
  • You don't have to remember everything!
  • You're not screwed if team members change unexpectedly
  • You don't have to reinvent (or reverse engineer) the wheel next time you do something similar
  • Appropriate Documentation - living documentation - a means to an end

Your assignment for WDD3 is essentially about documenting the analysis and design stages of your Web site.
