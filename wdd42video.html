Adding Sound and Video

Sound and video can be effective, however they must be used with care.

  • Primary purpose/visitor
  • Attraction or Detraction
  • Accessibility
  • Bandwidth/filesize vs Quality

There is no single solution that will suit all.

Where to get Sound

Recordings will be copyright waived or you/client hold the copyright for.

Sound clips will (almost certainly) have come from:

  • Live input using a Mic
  • CD/DVD/DAT/MiniDisc etc

Sound clips are usually captured onto a PC through the "line in" on the sound card, straight from a CD using the CD-ROM drive (or DVD using a DVD-ROM drive) or another digital input.

Where to get Video

Video footage is likely to have come from one of the following sources:

  • Digital Video Camera
  • WebCam or Digital camera
  • TV Capture

Video capture is usually performed using a Video capture card or FireWire and associated software (WebCams and digital cameras often use USB instead).

Compression

However video or sound files are captured on to your computer one of the key things to consider is compression.

Adding Sound

Sound is commonly distributed in one of the following formats:

  • Wav
  • Real Audio
  • Midi - uses built in "voices" from your sound card
  • MP3 - CD quality. Filesize roughly 1Mb per minute
  • WMA - Windows Media Audio
  • Ogg Vorbis - free, open, and unpatented

Adding sound 2

How to add sound to your pages:

  1. Downloadable sound clips
  2. Embedded sound clips
  3. Streaming Audio [Further info]

The visitor will need relevant software or a plug-in to be able to view the file and you should provide a link to the relevant software/plug-in.

Downloadable Sound Clips

Advantages:

  • Interoperability - a standard link so it will always work
  • Unobtrusive - visitor isn't forced to download the file
  • Flexibility - you can provide several different file types
  • Description - easy to give a description of the file and file size

Downloadable Sound Clips 2

Disadvantages:

  • You can't be sure a visitor will download it
    (why force people to download it though?)
  • Users may need to wait for the download

HTML code for including a sound file as a downloadable link:

<a href="example.mp3">30 seconds of Ian singing, 500K MP3 file</a>

Embedded Sound Clips

Advantages:

  • Potentially all visitors (with compatible software) will hear the sound
  • Can give an "ambience" to different areas of your site

Embedded Sound Clips 2

Disadvantages:

  • Visitors don't have a choice/may not work
  • Uses non-standard code
  • Bandwidth
  • Intrusive/annoying - the music may not be to your taste

Syntax for Embedding Sound

For completeness the HTML code to embed sound is as follows:

<embed src="/song/mesinging.mid" height="50" width="50">

Height and width are the dimensions of a toolbar allowing people to control playback. It is possible to use hidden="true" - this hides the toolbar and means visitors have no control over the music - please don't do this!
Autostart="true" - starts the sound automatically
loop="true" - loops the sound

Adding Video

You have two choices for making video available on the web:

  1. Progressive Streaming (HTTP - Hyper Text Transfer Protocol)
  2. Real Time Streaming (RTSP - Real Time Streaming Protocol)

Progressive Streaming

Not true streaming. The video is downloaded to the viewer's machine but the clip may begin playing before download is complete. User cannot skip ahead.

It is preferable where bandwidth is likely to be low but this can be problematic for long clips due to their file size.

Common formats used for downloadable video are:-

  • AVI
  • QT
  • MPG

Progressive Streaming

Two choices:

  1. Embedded
  2. Downloadable

Whichever you choose the visitor will need relevant software or a plug-in to be able to view the file and you should also provide a link to a source of the relevant software/plug-in.

Flash Video

Popular user generated video resources (such as YouTube and Google Video) use Flash Video that is progressively streamed.

In the last year the amount of Flash Video (FLV) available online has almost doubled to 22%. The report also stated that 97% of User Generated Video (UGV) is in Flash Video format, suggesting that the growth in FLV is likely to continue.

Embedded Video

<embed src="/video/example.mov" height="100" width="50" controller="false" autoplay="true" pluginspage="http://www.apple.com/" loop="true">

Height/width - set the size of your movie (pixels or percent)
Controller - adds controls for the movie (true/false)
Autoplay - If "true" your movie will play as soon as it starts to download, if "false" the user must click to play it.
Pluginspage - location of the required plug-in.
Loop - If "true" movie loops, if "false" movie plays once and stops, if "palindrome" movie plays to the end, plays back in reverse, plays to end again etc.

Real Time Streaming

Real Time Streaming Protocol (RTSP) better for long clips and on LAN or broadband/high speed environments.

Always requires a streaming server (additional expense).

Some ISPs offer a Real streaming server with their Web space.

Real Networks offers a free version of its new Helix server.

DreamHost offers the Darwin streaming server as part of their standard hosting package.

Real Time Streaming 2

The benefits:-

  • Plays automatically
  • Don't need to download whole clip before playing
  • The clip always resides on the server (can't be downloaded)
  • The visitor can jump to any part of the video without having to download the preceeding footage
  • Tries to keep pace with user's connection speed, server tries to maintain a constant connection
  • Video for streaming does need careful preparation

Real Time Streaming 3

Common formats are:-

  • Apple MOV
  • Microsoft WMV (was ASF)
  • RealNetwork's RM
  • Ogg Theora - free, open, and unpatented

Video streaming using the Flash player is becoming increasingly common. This does require video to be correctly encoded and access to a Flash Media Server or equivalent (e.g. Red5).
