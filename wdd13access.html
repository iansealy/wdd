Why Accessibility Is Important

  • 10% to 20% of the world's population suffer from disability
  • Is increasing as average age of population increases
  • Many of these disabilities affect access to the Web (e.g. visual, aural, dexterity and memory problems)
  • For example, blind users can't see graphics and users with neurological difficulties might be confused by complex presentation
  • Everyone has a right to access the Web

It's The Law

  • In the UK there is a legal obligation due to the Disability Discrimination Act
  • Most countries have disability discrimination laws
  • Numbers make commercial sense
  • >1 million visually impaired people in the UK
  • Developing a Web Accessibility Business Case for Your Organization
  • Convincing clients (and managers?)

Why Accessibility Is Important To All Users

Accessible Web sites can be used in a variety of environments:

  • Low bandwidth, where images are slow to download (because accessible sites are usable without images)
  • Noisy, where difficult to hear audio
  • Bright, where difficult to see screen
  • Mobile phone, where display is small
  • Driving, where eyes and hands are busy

Priorities

From Checklist of Checkpoints for Web Content Accessibility Guidelines:

  • Priority 1 - "A Web content developer must satisfy this checkpoint. Otherwise, one or more groups will find it impossible to access information in the document. Satisfying this checkpoint is a basic requirement for some groups to be able to use Web documents."

Priorities

  • Priority 2 - "A Web content developer should satisfy this checkpoint. Otherwise, one or more groups will find it difficult to access information in the document. Satisfying this checkpoint will remove significant barriers to accessing Web documents."
  • Priority 3 - "A Web content developer may address this checkpoint. Otherwise, one or more groups will find it somewhat difficult to access information in the document. Satisfying this checkpoint will improve access to Web documents."

Priority 1

  • Use text equivalent for all non-text content (e.g. by using alt attribute for images), including images, audio, video, animations, etc...
    (Use alt="" if image isn't important)
  • Ensure documents can be read without stylesheet (won't look the same, but will be readable) (zap style sheets bookmarklet or Firefox Web Developer extension)
  • Ensure information conveyed with colour is available without colour

More Priority 1

  • Avoid causing screen to flicker
  • Use clearest and simplest language appropriate for site's content (mainly because of users with cognitive problems, but also because users tend to skim read Web pages)
  • Use <th> to identify header cells (not <td><b>)
  • Ensure pages are usable when scripts are turned off
  • If page can't be made accessible then provide a link to an alternative page that is accessible

Priority 2

  • Ensure foreground and background colour provide sufficient contrast (important in bright environments, not just for users with poor vision)
    Low contrast High contrast
  • Use markup rather than images if possible
  • Use valid HTML
  • Use stylesheets

More Priority 2

  • Use relative rather than absolute units
  • Use headings to convey document structure
  • Mark up lists properly
  • Use <blockquote> for quotations, not to indent text
  • Don't make content blink
  • Don't automatically refresh page
    <meta http-equiv="Refresh" content="30">
  • Don't automatically redirect page (except on the server)
    <meta http-equiv="Refresh" content="30; URL=http://google.com/">

More Priority 2

  • Don't open new windows or popups without warnings
  • Include a site map or table of contents (useful for all users, but especially those with cognitive difficulties)
  • Use navigation tools consistently
  • Don't use tables for layout unless the tables make sense when linearised

Priority 3

  • Expand abbreviations and acronyms where they first occur
  • Until browsers render adjacent links distinctly include characters between adjacent links
    Search Engines
  • Provide navigation bars
  • Provide different search functionality for different skill levels
  • Provide a means to skip over ASCII art
  • Have consistent user interface
  • Provide summaries for tables
