WYSIWYG Editors

Kieren Pitts

Introduction

A popular way of producing Web pages

  • What is a WYSIWYG Editor?
    • What You See Is What You Get
  • Desktop applications:
    • Adobe Dreamweaver
  • "Through the Web" applications:
    • CKEditor
    • TinyMCE
    • Guardian Scribe

The Benefits of WYSIWYG Editors

  • You know what the page will look like before it is viewed in a browser (sort of...)
  • Simplicity:-
    • HTML code is generated for you
    • Easier to spot mistakes in content
  • Speed:-
    • Can be quicker to make major changes or if starting from scratch
    • Quicker to create complex code, such as tables
  • Integrated site management tools
  • Concentrate on design rather than the code

Problems with WYSIWYG Editors

  • What You See is ALMOST What You Get
  • Don't always produce valid code
  • Can include editor-specific code
  • Often produce inefficient code
  • Problems can occur if you add extra code (e.g. by using Notepad or Brackets) and then load the same file back in to the WYSIWYG editor

More Problems with WYSIWYG Editors

  • Desktop WYSIWYG software
    • can be expensive
    • can require a powerful PC
    • often tied to a particular PC so you can't make updates remotely
  • To keep up-to-date with Web standards you need to upgrade and manufacturers often can't/don't keep up - hard to justify the purchase?
  • Prevent developers building up core skills
    • Developers learn an application without understanding the language
      • Easy to spot people who can hand code
      • Developers work with what's available in their application and not what's possible in browsers
    • Hand coders usually more knowledgeable and more beneficial to an organisation
      • Code is more efficient
      • Developers are better skilled at finding and fixing errors/issues
      • Developers have a better understanding of underlying languages
      • Developers can fix things or make changes when away from their desk

Desktop WYSIWYG editors have had their day

  • The Web has changed:
    • Edited using "through the Web" tools and a CMS
    • Distributed editors or community/user generated content
    • Web site comprise a suite of technologies, not just HTML and CSS
    • The 'one package fits all' approach doesn't match current working practices.
      Project teams often comprise:
      • Web designer
      • HTML coder
      • Front-end interaction designer
      • Back-end developer
      • Content editor
  • Talk of the decline of WYSIWYG editors has been going on for some time:
    • The end of WYSIWYG editors
    • I'm sorry but Dreamweaver is dying... Dreamweaver is still dying
    • Most significantly, Dreamweaver is a declining trend in job adverts
  • Adobe produce Brackets
  • Macaw was a new desktop WYSIWYG editor, launched in 2014 but development ceased in 2016.
