WYSIWYG Editors

WYSIWYG Editors

Kieren Pitts

Introduction

Arguably the most popular way of producing Web pages

  • What is a WYSIWYG Editor?
    • What You See Is What You Get
  • Desktop applications:
    • Microsoft Expression Web
    • Adobe Dreamweaver - was Macromedia
    • Adobe GoLive - discontinued 2008
    • Microsoft FrontPage - discontinued in 2006
  • "Through the Web" applications:
    • TinyMCE
    • CKEditor (was FCKEditor)
    • YUI 2 - Rich Text Editor

The Benefits of WYSIWYG Editors

  • You know what the page will look like before it is viewed in a browser (sort of...)
  • Simplicity
    • HTML code is generated for you
    • Easier to spot mistakes in your content
  • Speed
    • Major changes or starting from scratch
  • Site management
  • You can concentrate on design

Problems with WYSIWYG Editors

  • WYSIAWYG
    • What You See is ALMOST What You Get
  • Dreamweaver in action
  • They don't always produce valid code
  • They produce editor-specific code
    • FrontPage is the worst... by a long, long way! Thankfully Expression Web is much better.
    • They try to be too clever
  • They often produce inefficient code
  • Problems can occur if you add extra code (e.g. by using Notepad)

More Problems with WYSIWYG Editors

  • They are "bigger" packages
    • Resources
    • Cost
  • To keep up to date with the Web standards you need to upgrade and manufacturers often can't/don't keep up
  • They can actually remove the creative thought processes behind designing Web pages
    • Themes (FrontPage)
  • Prevent developers building up core skills
