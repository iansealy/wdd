WYSIWYG Editors

WYSIWYG Editors

Kieren Pitts

Introduction

Probably the most popular way of producing Web pages

  • What is a WYSIWYG Editor?
    • What You See Is What You Get
  • Examples:
    • Microsoft FrontPage - now discontinued
    • Microsoft Expression Web
    • Adobe GoLive
    • (Macromedia) Adobe Dreamweaver

Who Would Use a WYSIWYG Editor?

  1. Someone who didn't know any HTML
  2. Someone who didn't want to learn any HTML
  3. Someone that's familiar with HTML and wants to take advantage of the benefits offered by a WYSIWYG editor

The Benefits of WYSIWYG Editors

  • You know what the page will look like before it is viewed in a browser (sort of...)
  • Simplicity
    • HTML code is generated for you
    • Easier to spot mistakes (in layout or spelling etc...)
  • Speed
    • Major changes or starting from scratch
  • Site management
  • You can concentrate on design

Problems with WYSIWYG Editors

  • WYSIAWYG
    • What You See is ALMOST What You Get
  • Dreamweaver in action
  • They don't produce valid code
  • They produce editor-specific code
    • FrontPage is the worst... by a long, long way!
    • They try to be too clever
  • They often produce inefficient code
  • Problems can occur if you add extra code (e.g. by using Notepad)

More Problems with WYSIWYG Editors

  • They are "bigger" packages
    • Resources
    • Cost
  • To keep up to date with the Web standards you need to upgrade and manufacturers often can't keep up
  • They can actually remove the creative thought processes behind designing Web pages
    • Themes (FrontPage)
    • Wizards (FrontPage AGAIN)

[WYSIWYG editors] were designed to solve a 1994 problem.
