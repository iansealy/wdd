Content Management Systems (CMS)

Kieren Pitts

What is a Content Management System?

Working definition:

Web-based software used to publish and maintain content on a Web site.

  • Web-based software - log in and make changes just using your browser
  • Maintaining content - design work done separately
  • Not always used for the whole site

    • There are literally hundreds of CMS tools but the big three are: WordPress, Drupal and Joomla

Work flow comparison - hand coding

Creating a new page...

Hand coding:

  1. Create HTML file locally by marking up content
  2. Edit other files locally to create links to your new page
  3. Update SSIs to include page in nav or sub-nav
  4. FTP the changed, and new, files (including any images) to your web space

Work flow comparison - using a CMS

Kieren's description of using a CMS:

  1. Log in to software hosted on server
  2. Paste/type in your content using online editor (usually WYSIWYG)
  3. Upload any images (often using the same editor)
  4. Nav/sub-nav links generated
  5. Add links from other pages

Common features of a CMS:

  • Create and maintain content through the web
    • No HTML/CSS skills needed to edit content (normally the major selling point from the perspective of a manager)
    • Only edit the page content
  • Storage of content (usually in a database):
    • Some linking can be automatic
    • Nav elements and site maps are automatically generated
  • Edit content whenever you have an Internet connection
  • Preview page before publishing
  • Templating

Common features of a CMS continued:

  • Version control/work flow/audit trails/roll back
  • Access control - only certain users can access a specific area
  • Site search
  • User registration and commenting
  • Plugins/extensibility
    • 22,000+ Drupal modules

Downsides of a CMS

  • Software tie-in
    • Upgrades and patches
    • What about plugins/extensions?
    • Will the CMS last?
  • Requires a database
    • Harder to keep local/dev copy in sync
    • Hosting cost
    • Performance
    • Back-ups
  • Requires an Internet connection
  • Management of content other than web pages not so good (e.g. PDFs)
  • Configuration can be time consuming

Downsides of a CMS continued

  • They can be very complicated
  • The sites look the same
  • Sometimes produce 'ugly' URLs (for search engines)
  • CMSs do lots of things well but rarely do everything you want:
    • Find a different CMS
    • Look for a module or plugin
    • Write the code yourself
  • You can outgrow the software:
    • iSpot and Drupal
    • Requirements change
    • 'One size' does not fit all

How are you likely to work with a CMS?

The brief: Your client wants to be able to update their website after you've delivered it.

  • You install and set up the CMS for a client
    • You may need to explain the hosting requirements and extra cost to the client
  • You convert your design (or someone else's) into the templates required by your CMS

Converting your design

  • You need to understand the HTML, CSS and JavaScript
  • You need to understand the templating system of your chosen CMS
  • You will often need to make changes to the mark-up to include CMS specific code (for the nav etc)
  • You may need different templates for different sections of the site
  • You may need to write some code (e.g. PHP) to meet the client's requirements
  • This is where your hand-coding, design and programming skills come in!

Content upload and delivery

  • You may do the initial content upload
  • You may train the client in using the CMS
  • At some point the day-to-day running is handed over to the client
  • Maintenance:
    • Doing upgrades and installing patches
    • Adapting the templates for new site sections
    • Adapting the templates when the client decides they want a different style of page

Retrofitting a CMS

  • You have a static site you want to maintain using a CMS
    • Create your templates
    • Update your mark-up and CSS to work with the CMS
    • Move content - populate the right content areas in the template
    • Move all the other files (anything that's not a web page)
  • Some of the lighter weight CMSs make migration easier:
    • Perch
    • CushyCMS
  • Often the best approach is to just start afresh:
    • Good time for a clean up
    • New design/new IA

Free templates and themes

Doesn't just apply to CMSs

  • You still need coding skills:
    • Assess the quality of the code (semantics and accessibility)
    • Convert template for use on your site/CMS
  • The design may not match requirements of client
    • User needs analysis
    • Discussion with stake-holders
    • User testing
  • Another site may use the same template

WordPress

  • Started life as a blogging tool in 2003
  • Developed into a feature-rich CMS
    • Current version: 3.5.1 (Jan 2013)
  • Used by over 14% of "top 1 million" websites (Alexa)
  • Over 25,000 plugins
  • Over 1,700 themes
  • WordPress.com - hosted blogging service
  • WordPress.org - blogging software
  • Famous 5-minute installation

Drupal

  • Launched in 2001
  • Latest version 7.22 (April 2013)
  • Accounts for 2.1% of all websites worldwide
  • 20,000+ contributed modules
  • Sophisticated API for developers
  • Drupal 8 uses core components from the Symfony2 framework
  • Drupal.org
