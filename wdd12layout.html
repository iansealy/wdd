Using Tables To Control Layout

  • Technically, tables should only be used to present tabular data (e.g. a profit and loss account)
  • HTML was not designed for layout
  • Layout is best done with CSS (cascading stylesheets), but lots of sites (especially old ones) still use tables
  • Situation is rapidly changing though - almost all new sites use CSS Positioning
  • Will look at CSS-P later in course

Pros and Cons

  • Alternatives to tables (i.e. CSS) don't work in really old browsers
  • If all content is in one cell then must wait for entire page to load before displays
  • Old browsers render tables very slowly
  • Makes content less accessible to, e.g, text-to-speech readers and mobile phones
  • Table-based layouts are harder to maintain
  • CSS layouts are less verbose (ESPN.com saved two terabtyes of bandwidth per day by switching to CSS)
