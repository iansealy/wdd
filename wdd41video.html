Adding Sound and Video

Sound and video can be effective, however they must be used with care.

  • Primary purpose/visitor
  • Attraction or detraction
  • Accessibility
  • Bandwidth/file size vs Quality

There is no single solution that will suit all.

Where to get Sound

Recordings will be copyright waived or you/client hold the copyright for.

Sound clips will (almost certainly) have come from:

  • Live input using a Mic (e.g. podcasting)
  • CD/DVD/DAT/MiniDisc etc

Sound clips are usually captured onto a PC through the "line in" on the sound card, straight from a CD/DVD using the CD/DVD drive or another digital input.

Where to get Video

Video footage is likely to have come from one of the following sources:

  • Digital Video Camera
  • WebCam or Digital stills camera
  • Mobile phone
  • TV Capture

High quality video capture is usually performed using a dedicated video capture card (or FireWire) and associated software. Lower quality video can be captured via USB or internal cameras in laptops/phones etc.

Compression

However video or sound files are captured on to your computer one of the key things to consider is compression.

Adding Sound

Sound is commonly distributed in one of the following formats:

  • Wav
  • Real Audio
  • Midi - uses built in "voices" from your sound card
  • MP3 - CD quality. File size (very) roughly 1Mb per minute
  • WMA - Windows Media Audio
  • Ogg Vorbis - free, open, and unpatented

Adding sound 2

How to add sound to your pages:

  1. Downloadable sound clips
  2. Embedded sound clips (using an HTML tag or Flash player)
  3. Streaming Audio (covered in streaming video)

The visitor will need relevant software or a plug-in to be able to view the file and you should provide a link to the relevant software/plug-in.

Downloadable Sound Clips

Advantages:

  • Interoperability - a standard link so it will always work
  • Unobtrusive - visitor isn't forced to download the file
  • Flexibility - you can provide several different file types
  • Description - easy to give a description of the file and file size

Downloadable Sound Clips 2

Disadvantages:

  • You can't be sure a visitor will download it
    (why force people to download it though?)
  • Users may need to wait for the download

HTML code for including a sound file as a downloadable link:

<a href="example.mp3">30 seconds of Ian singing, 500K MP3 file</a>

Embedded Sound Clips

Embedding using the <embed> tag.

Advantages:

  • Potentially all visitors (with compatible software) will hear the sound
  • Can give an "ambience" to different areas of your site

Embedded Sound Clips 2

Disadvantages:

  • Visitors don't have a choice/may not work
  • Uses non-standard code - your pages won't validate
  • Bandwidth
  • Intrusive/annoying - the music may not be to your taste or the user might be listening to something else already

Syntax for Embedding Sound

For completeness the HTML code to embed sound is as follows:

<embed src="/song/mesinging.mid" height="50" width="50">

Height and width are the dimensions of a toolbar allowing people to control playback.
It is possible to use hidden="true" - this hides the toolbar and means visitors have no control over the music - please don't do this!
Autostart="true" - starts the sound automatically
loop="true" - loops the sound

Video delivery

You have two choices for making video available on the web:

  1. Progressive Streaming (HTTP - Hyper Text Transfer Protocol)
  2. Real Time Streaming (RTSP - Real Time Streaming Protocol)

Progressive Streaming

Not true streaming. The video is downloaded to the viewer's computer but the clip may begin playing before download is complete. User cannot skip ahead unless that part of the movie has already downloaded.

For example: The hunt for Gollum trailer

It is preferable where bandwidth is likely to be low but this can be problematic for long clips due to their file size.

Common formats are:-

  • AVI
  • QT
  • MPG
  • FLV

Progressive Streaming

Two choices:

  1. Embedded (using an HTML tag or Flash player)
  2. Downloadable

The visitor will need relevant software or a plug-in to be able to view the file and you should also provide a link to a source of the relevant software/plug-in.

Embedded Video

Embedding using the <embed> tag.

<embed src="/video/example.mov" height="100" width="50" controller="false" autoplay="true" pluginspage="http://www.apple.com/" loop="true">

Height/width - set the size of your movie (pixels or percent)
Controller - adds controls for the movie (true/false)
Autoplay - If "true" your movie will play as soon as it starts to download, if "false" the user must click to play it.
Pluginspage - location of the required plug-in.
Loop - If "true" movie loops, if "false" movie plays once and stops, if "palindrome" movie plays to the end, plays back in reverse, plays to end again etc.

Real Time Streaming

Real Time Streaming Protocol (RTSP) better for long clips and on LAN or broadband/high speed environments.

Always requires a streaming server (additional expense/expertise).

Some ISPs offer a streaming server with their Web space.

Real Networks offers a free version of its Helix server.

DreamHost offers the Darwin streaming server as part of their standard hosting package.

Real Time Streaming 2

The benefits:-

  • Plays automatically
  • Don't need to download whole clip before playing
  • The clip always resides on the server (can't be downloaded easily)
  • The visitor can jump to any part of the video without having to download the preceding footage
  • Tries to keep pace with user's connection speed, server tries to maintain a constant connection
  • Video for streaming does need careful preparation
  • Video streaming can require greater processing power on the client machine

Real Time Streaming 3

Common formats are:-

  • Apple MOV
  • Microsoft WMV (was ASF)
  • RealNetwork's RM
  • Flash video - FLV
  • Ogg Theora - free, open, and unpatented

Video streaming using the Flash player is becoming increasingly common. This does require video to be correctly encoded and access to a Flash Media Server or equivalent (e.g. Red5).

Linking to Streaming Media

<a href="example.avi">Streaming lecture example</a>

This causes an external player to open and connect to the streaming server, the player will depend on the format of the video. Common players include:

  • Windows Media Player
  • QuickTime Player
  • RealPlayer
  • Flash Player

Can also embed streamed video, e.g. Biz/ed or the BBC

Flash Video

Popular user generated video resources (such as YouTube and Vimeo) use Flash Video.

Both services originally used progressive streaming but now appear to use real time streaming for many videos.

Between 2005 and 2006 the amount of Flash Video (FLV) available online almost doubled to 22%. The report also stated that 97% of User Generated Video (UGV) is in Flash Video format, suggesting that the growth in FLV is likely to continue.

Both services have HTML code you can use to embed videos but it's not always standards compliant.

Accessibility

There are many accessibility issues associated with providing audio and video online. Some of the issues to consider include:

  • Which assistive technologies support the player you've chosen?
  • Does the player have keyboard controls or is it mouse only?
  • Providing transcripts
  • Providing captions (subtitles for video)

In general, no single solution will be accessible so providing the same information is a variety of forms is recommended. For example: Biz/ed
