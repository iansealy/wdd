User testing

User testing

It's all about evaluation

User testing is powerful!

  • Usability testing with real users is the most fundamental and useful usability method
  • Methods range from informal testing to precisely controlled studies
  • Testing may be quantitative or qualitative

Test goals and test plans

  • What do you want to achieve?
  • Who will run the test?
  • Who will the test subjects be?
  • What tasks will they be asked to perform?
  • How will we measure or record their performance?
  • How much can the experimenter help the user?
  • How will the data be analysed? (statistics!)
  • What will the criteria be for pronouncing the usability of the site to be a success?

Getting test users

  • Should be as representative as possible
  • For an intranet, it's easy to find appropriate users!
  • For an Internet site, need to have a sample of users with similar demographic distribution to the intended user population
  • Jakob says you only need to test with 5 users, but not everyone agrees!

Test tasks

  • Should be as representative as possible of the uses to which the system will be put
  • Should provide reasonable coverage of the most important parts of the site
  • Can be based on task analysis information previously collected
  • Sometimes may be preferable to allow users to design their own tasks

Measuring performance (Quantitative testing)

Needs to be relevant to task of interest! Typical usability measurements include:

  • time to complete a task
  • number of mouse clicks required to complete a task
  • ratio between successful interaction and errors
  • number of errors
  • number of system features a user can remember
  • amount of "dead" time (e.g. system response time and "thinking" time)

These are the things that can help you to measure ROI

Think aloud protocol (Qualitative testing)

  • Extremely useful usability method
  • Less formal
  • Participant tests the system whilst continuously thinking out loud
  • Enables us to understand their thought processes and identify misconceptions (mental models again!)
  • BUT doesn't allow performance measurement to be carried out
  • Also, can be unnatural for many people

Debriefing

  • User debriefed after test and asked to fill in any subjective satisfaction or general usability surveys (the System Usability Scale is a particularly useful and flexible one)
  • Asked for comments about the system and suggestions for potential improvements

Tasks

Doing a formal user test takes a lot of time, so we're going to do a couple of brief DIY tests:

Task 1 (if you have a Hotmail account)

  • Go to www.mesh.com and sign up for Microsoft's new MESH service. If you can't sign up, register your interest in the service instead.

Test 2 (everyone)

  • Go to honda.co.uk
  • Find out the mpg of the hybrid version of the Honda Civic

Before starting, think about (and discuss) how you might quantify the success of your tasks.

At the end of the test, discuss your feelings about the task and the Honda site in general with the person sitting closest to you.
