What is Flash?

  • Flash started life as a basic animation tool (called Futuresplash - bought by Macromedia)
  • It is now a complex programming environment
  • It's a vector-based multimedia authoring package
  • Timeline based
  • Online or offline?
  • Proprietary format (needs a plug-in/player)
  • Macromedia now owned by Adobe

The Trendy - what is Flash used for?

  • Intros [Jazz Aspen]
  • Whole sites [thevoid]
  • Interactive Games [American Dad vs Family Guy kung fu]
  • Animation/Comics [DC Comics]
  • Education [Biz/ed]
  • Advertising [Rich Media Resources]
  • Offline multimedia

The Bad

What is bad about Flash?

  • It is proprietary
    • Macromedia say that 98% of European Web users can view Flash content
    • Flash 6 penetration is 98.5% but the latest is v9 (83.5%)
    • For a site getting 1,000,000 visitors a month at least 15,000 (1.5%) people can't view Flash 6 content and 165,000 can't view Flash 9 content
  • Poor (but improving) accessibility (text-to-speech, people who can't use a mouse)
  • Limited support for other Web-enabled devices

The Bad - Part 2

  • Flash Intros
  • Gratuitous animation [Example]
  • Forces users to take a particular direction, can't link to it directly
  • Non-standard GUIs
  • Disables the Back button [Example]
  • It can't be searched
  • Text size cannot be changed and non-standard fonts are used

The Bad - Part 3

  • Updates are harder
    • It's developed using specific tools
    • The files used on Web sites cannot be edited
  • The development environment costs money
  • Printing a page is difficult
  • Flash content can be enormous
  • Can't view source
  • It means learning a new language (if you want to be able to do the clever stuff)

The Good

  • Currently the best way to bring true, real-time interactivity to the Web
  • It is a fully-featured programming environment
  • It has reasonable connectivity to other technologies e.g. XML
  • Animation and Sound
  • Precise positioning is possible
  • It's vector-based
  • Filesize

The Good - Part 2

  • Does have some 'good' Accessibility points - e.g. Zoom
  • With each release more accessibility issues are resolved
  • If your content must be animated or interactive (i.e. you're an animator)
  • You can't view source
  • It's here to stay

Incorporating Flash content

  • Does my site need it?
  • Can I present the information in a more accessible/usable way?
  • Can I afford to lose 1.5 - 16.5% of my visitors/income?
  • Is it cost effective? Flash development is expensive
  • Can I afford the time to learn how to use Flash and how to program (in ActionScript)?

Incorporating Flash - Part 2

Flash will create the code needed to embed a SWF file in a Web page for you:

<object classid="clsid:D27CDB6E-AE6D-11cf-96B8-444553540000" codebase="http://download.macromedia.com/pub/shockwave/cabs/flash/swflash.cab#version=6,0,0,0" width="100%" height="100%" id="example" align="">
<param name="movie" value="example.swf">
<param name="quality" value="high">
<param name="bgcolor" value="#FFFFFF">
<embed src="example.swf" quality="high" bgcolor=#FFFFFF width="100%" height="100%" name="example" align="" type="application/x-shockwave-flash" pluginspage="http://www.macromedia.com/go/getflashplayer"></embed> </object>
