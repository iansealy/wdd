What is Flash?

  • Flash started life as a basic animation tool called FutureSplash in 1995
  • Bought by Macromedia in 1996 and renamed Flash
  • It's a vector-based multimedia authoring package
  • Time line based
  • Used online and offline
  • Proprietary format (needs a plug-in/player)
  • It's become a complex programming environment
  • Adobe bought Macromedia in 2005

The Trendy - what is Flash used for?

  • Intros [Hema shop]
  • Whole sites [thevoid]
  • Interactive Games [Zombie Cricket]
  • Animation/Comics [The Spider Cliff Mysteries]
  • Education [Biz/ed]
  • Advertising [Rich Media Resources]
  • Video/audio players [BBC]
  • Rich Internet Applications (RIA) - can be desktop applications [Adobe AIR]

The Bad

What is bad about Flash?

  • It is proprietary
    • Adobe say that 99% of European Web users can view Flash content
    • Flash 9 (and below) player penetration is 99.3% but the latest is version 11 and that's not listed. Player v10.3 has a penetration of (40.5% - 51.5%)
    • For a site getting 1,000,000 visitors a month at least 3,000 (0.3%) people can't view Flash 9 content and 595,000 can't view Flash 10.3 content
  • Accessibility - it's harder to make Flash content accessible than it is an HTML page
  • No support on some important devices such as iPhone and iPad

The Bad - Part 2

  • Flash Intros [Example]
  • Gratuitous animation [Example]
  • Forces users to take a particular direction, can't link to it directly
  • Non-standard GUIs
  • Disables the Back button [Example]
  • It can't be searched easily (although support is improving even in Google Preview)
  • Text size cannot be changed and non-standard fonts are used

The Bad - Part 3

  • Updates are harder
    • It's developed using specific tools
    • The files used on Web sites cannot be edited
  • The development environment costs money
  • Printing a page is difficult
  • Flash content can be enormous
  • Can't view source
  • It means learning a new language (if you want to be able to do more than basic animation)

The Good

  • Currently the best way to bring true, real-time interactivity to the Web
  • It is a fully-featured programming environment
  • It has reasonable connectivity to other technologies e.g. XML
  • Animation, video and sound
  • Precise positioning is possible
  • It's vector-based
  • Filesize

The Good - Part 2

  • Does have some 'good' accessibility points - e.g. zoom (although this is superseded in modern browsers)
  • With each release more accessibility issues are resolved
  • If your content must be animated or interactive (i.e. you're an animator or games designer)
  • Almost ubiquitous multimedia delivery
  • You can't view source

Generating Flash content

  • Authoring is done using a specific application
  • The application creates .fla (or .xfl) files
  • When a Flash file is finished it's compressed and optimised before being used online
  • Optimised files have a .swf file extension
  • Once optimised, the files are difficult to edit. The .fla files should be retained

Incorporating Flash content

  • Does my site need it?
  • Can I present the information in a more accessible/usable way?
  • Can I afford to lose some of my visitors/income?
  • Is it cost effective as Flash development is expensive
  • Do I want to invest the time to learn how to use Flash and how to program in ActionScript?

Flash and HTML5 - 1

  • iPad, iPod Touch and iPhone don't/won't support Flash
  • Is it about openness, CPU load, security/reliability or a walled garden?
  • The HTML5 standard isn't finalised and support is patchy but improving
  • Flash is not based on open standards
  • HTML5 and Flash can both be CPU hogs
  • Flash has had some high profile security issues
  • Flash can cause instability in browsers
  • Other services/tools allow Flash content to be played on iPads etc (iSwifter or Skyfire)
  • Authoring tools are key

Flash and HTML5 - 2

  • Flash players may be replaced by native support for audio/video (covered in WDD42)
    • <audio>
    • <video>
  • <canvas>
    • An empty container that you can use for drawing or animation using JavaScript.
    • Support for:
      • Common shapes (circles, rectangles etc.)
      • Arcs and curves
      • Fills, strokes and shadows
      • Gradients
      • Transparency
      • Transformations (scaling, rotation, translation)
      • Text
      • Import other images or data

<canvas> element

<canvas id="mypicture" width="150" height="150"></canvas>

The canvas element can only be drawn in using JavaScript

<script type="text/javascript">
var c=document.getElementById("mypicture");
var cxt=c.getContext("2d");
cxt.fillStyle="#0D8DAD";
cxt.beginPath();
cxt.arc(75,75,30,0,Math.PI*2,true);
cxt.closePath();
cxt.fill();
</script>
