Table Tags

Every table has at least three or four tags:

  • <table> - defines table (one per table)
  • <tr> - defines a table row (one per row)
  • <th> - defines a header cell
  • <td> - defines a data cell

<table> tags enclose the whole table. <tr> tags enclose each row. Each set of <tr> tags will contain one or more <th> or <td> tags.

Simple Example

<table>
    <tr>
        <th>Team</th>
        <th>Points</th>
    </tr>
    <tr>
        <td>Man City</td>
        <td>20</td>
    </tr>
    <tr>
        <td>Arsenal</td>
        <td>20</td>
    </tr>
    <tr>
        <td>Liverpool</td>
        <td>20</td>
    </tr>
</table>

table Attributes

  • border (in pixels)
  • cellspacing (in pixels)
  • cellpadding (in pixels)
  • width (in pixels or percent)
  • align (left, right or center)
  • summary (text description of the table)

cellpadding is the space between the content of the cell and the edge of the cell. cellspacing is the space between each cell.

tr Attributes

  • align (left, right or center)
  • valign (top, middle, bottom)
  • bgcolor (background colour of row)

th And td Attributes

  • align (left, right or center)
  • valign (top, middle or bottom)
  • width (in pixels or percent)
  • bgcolor (background colour of cell)
  • colspan (merge columns)
    cell cell
    colspan="2"
  • rowspan (merge rows)

Using Tables To Control Layout

  • Technically, tables should only be used to present tabular data (e.g. a profit and loss account)
  • HTML was not designed for layout
  • Layout is best done with CSS (cascading stylesheets), but tables used to be used and often still are for HTML emails
