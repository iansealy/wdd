Prototypes

  • In addition to defining the site architecture, at this stage you will start creating prototypes and mock-ups of your interface for each individual Web page
  • You'll need to have sorted out your site architecture and navigation / classification / labelling before you can attempt this

Prototypes and iteration

You can never design a perfect site at the first attempt. Produce a prototype, test it (with real users) then refine it. Then do it again!

High fidelity prototypes / mockups

Pros:

  • Looks nice
  • Can be reasonably fast

Cons:

  • Visual design can detract from the interaction and information design
  • Modifications can be slow

Low fidelity prototypes/mockups (wireframes)

Wireframes /cont

  • Bare-bones descriptions of how the pages and components fit together
  • Simple line drawings
  • Each page is uniquely numbered (e.g. 2.1) and heavily annotated
  • Can have single wireframe if all pages follow the same pattern, but often need more (e.g. home page usually very different); on some sites every page will need a different wireframe
  • No graphic design

Wireframes: pros & cons

Pros:

  • Focuses on information design
  • Quicker than high fidelity mock-ups
  • Easy to change
  • Allows freedom for graphic design

Cons:

  • Clients often expect to have something that "looks nice"
  • Not very interactive (although can be made so)
  • Requires extra documentation

Functional specification deliverables

A functional specification is essentially a blueprint for a Web site. It is also for communicating detailed information about the goals and functional details of the site (e.g. to clients or developers).

Some or all of the following may be found in a functional specification document:

  • Business analysis
  • Competitor analysis
  • User analysis (may include personas)
  • Task analysis (may include scenarios)
  • Technical requirements
  • Site map / architecture
  • Workflows (interaction design)
  • Prototypes (mock-ups)

More information about how to write a specification (with examples) can be found at Mojofat.
