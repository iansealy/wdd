Caching

  • Fundamental problem is caching
  • Caching occurs at browser level (one person) and ISP level (many people)
  • Page may be downloaded once, but viewed many times (by one person or many people)

Visitors

  • Can't estimate number of visitors
  • Assumes 1 IP address = 1 visitor
  • But many users may appear to connect from the same IP address (because they're using the same proxy server)
  • Each request from an AOL user comes from a different IP address (multiple proxy servers)

Visits

  • Can't estimate number of visits
  • 1 "visit" = activity from 1 IP address that is terminated by 30 minutes of inactivity
  • Again, one IP address does not correspond to one visitor
  • Can be a 30 minute gap in a genuine visit (e.g. go to another site temporarily or make a cup of tea)
  • 30 minutes is entirely arbitrary

Cookie Tracking

  • Cookie tracking can improve estimates of visitors and visits
  • Cookies are short bits of text sent by a Web server that get stored on the local computer
  • Associated with a domain (e.g. microsoft.com)
  • Each time that domain is accessed the cookie is sent back to the Web server
  • In cookie tracking, if a cookie has not been set, then a cookie containing a unique ID is sent
  • Cookie gets sent back to server on each subsequent request
  • Unique ID is stored in server log

Cookie Tracking

  • Allows tracking of an individual
  • But some users refuse to accept cookies
  • Some might delete cookies after each session
  • Some browsers don't support cookies

False Information

  • Can't even trust browser type and referrer URLs
  • Many browsers can be configured to lie about their browser type and may do so by default
  • A few browsers send incorrect referrer URLs
  • Anonymisers can be used that deliberately send false browser types and referrer URLs

Other Problems

  • Can't determine a user's path through site (users often backtrack to cached pages)
  • Can't tell where entered site (home page is most likely to be cached, so first request will be for a deeper page)
  • Can't tell what site user went to next (because not logged, although could use custom script)
  • Can't tell how long was spent reading a page (user may have gone to another site imbetween or have made a cup of tea)
  • Can't tell how long was spent on site (same reason as above, plus don't know how long was spent on last page, which is likely to be long if they found something or short if became frustrated and left)

Remote Logging

  • If can't access server logs then often use remote logging
  • Involves embedding HTML for an inline image hosted by a third party
  • Third party counts accesses to this image and presents graphical report
  • Article and example reports
  • Relies on use of graphical browser
  • Ensure image is outside table structure of page or might delay rendering
  • ScriptSearch.com
