WDD Systems

Each of you has been given a username and password that allows you to access Birkbeck computers. If you forget your username then a list is available on the course Web site at:

 http://wdd0708.bbkweb.org/student.html

You should change your password from the one you have been allocated to one of your choosing as soon as possible. You can change your password at:

 http://www.bbk.ac.uk/ccs/password/

You will need your username and password in order to login to the computers in Room 402 at Birkbeck.

Network Drives

At Birkbeck you have two network drives, the N: drive and the Z: drive. When you're working during a session at Birkbeck, you should save all files in the N: drive or Z: drive. Both drives are available on any Birkbeck computer you login on. The contents will stay the same between sessions, but you are limited to 10 MB of files.

File Extensions

By default Windows hides file extensions, but this is annoying for Web development. For example, football.html and football.htm are different files, but will appear to be the same in Windows. So you need to always show all file extensions. This is generally done by opening Windows Explorer and then choosing Tools --> Folder Options --> View tab and then unchecking "Hide extensions for known file types". You'll want to make this change on your home computer as well as at Birkbeck.

Web Space

Each student has their own Web site on a DreamHost server. The URLs are all of the form http://username.wdd0708.bbkweb.org/. So, for example, the URL of my Web space is:

 http://iansealy.wdd0708.bbkweb.org/

A list of all the URLs is available from:

 http://wdd0708.bbkweb.org/student.html

Each student has a quota of 50 MB.

Your username is the same for Birkbeck and for DreamHost. The passwords were originally the same, but you need to manually keep them in sync. Changing your Birkbeck password will not change your DreamHost password (and vice versa). See above for changing your Birkbeck password. Changing your DreamHost password is more complicated and involves logging into a remote DreamHost server. This can be done using a program called PuTTY that's installed on the Birkbeck machines and can also be downloaded for free from:

 http://www.chiark.greenend.org.uk/~sgtatham/putty/download.html

Run PuTTY and then in the Host Name box type bbkweb.org and press the Open button. When the "login as:" prompt appears, type your username and press Enter. Then enter your password and press Enter. You should get logged into the Birkbeck server. Once you've logged in type passwd (in lower case) and then, when prompted, enter your existing password and press Enter. Then enter your new password (twice). If the system doesn't accept your new password then you'll need to run passwd again and retry. If the system accepts your new password then type exit and press Enter.

FTP

In order to put files on your Web site you will need to upload them to the DreamHost server. This is possible via FTP. FTP stands for File Transfer Protocol and simply allows you to transfer files from one computer to another.

There are scores of different FTP programs available, but they all follow the same basic scenario:

  • Specify remote computer to connect to
  • Give username and password
  • Either specify files to GET from the remote computer or PUT files from the local computer onto the remote computer

For accessing your DreamHost Web space, the computer you want to connect to is:

 bbkweb.org

You'll then need to login using your DreamHost username and password. Once you've been authenticated you'll be able to either GET files from the remote computer (the DreamHost Web server) or PUT files from your computer onto the DreamHost Web server.

FTP Clients

In order to access your Web space via FTP, you'll need an FTP client. For the handful of you that use Macs, the FTP program that I gather is good is Cyberduck, which you can find out more about from:

 http://cyberduck.ch/

Also, FileZilla (see below) has recently been made available for Macs, so you may like to try that out too.

If you're using Windows, then I recommend you install FileZilla, which is also on the Birkbeck computers. The current version can be downloaded from:

 http://filezilla-project.org/download.php?type=client

After you've downloaded FileZilla, you can install it by simply double clicking on it. On the first screen is a License Agreement, which you should agree to. The default options on the next few screens are all OK and you can just accept them.

Once FileZilla is installed, you can run FileZilla by choosing it from the Start menu. The FileZilla window is split into two main panes. The left-hand pane shows the files on your local computer. The right-hand pane shows the files in a directory on the remote computer. Since you haven't yet connected to a remote computer, the right-hand pane will be blank. Above these two main panes is a pane showing connection information and below is a pane showing the details of current transfers. Both of these panes will currently be blank. The simplest way to use FileZilla is to fill in three of the boxes at the top of the screen. In the "Host" box you should enter:

 bbkweb.org

In the "User" box you should enter your DreamHost username. If you forget your username then a list is available on the course Web site at:

 http://wdd0708.bbkweb.org/student.html

In the "Password" box you should enter your DreamHost password. Finally you click on the "Quickconnect" button. You should then connect to the DreamHost server and the files in your Web space will be visible in the right-hand pane of the FileZilla window.

If you want to transfer a file from your computer to the Web server then you should navigate to that file in the left-hand pane and then drag and drop it into the right-hand pane.

If you want to transfer a file from the Web server to your PC so that you can work on it then you follow the same procedure but in reverse. You should navigate to the file in the right-hand pane and then drag and drop it to the left-hand pane.

Web Site Structure

Any files that I upload to the htdocs directory of my Web space will be visible on the Web. For example, if I FTP a file called ian.html to the htdocs directoy then I will be able to view this file on the Web by appending the name of the file to my base URL and going to:

 http://iansealy.wdd0708.bbkweb.org/ian.html

If I create a subdirectory in htdocs called birkbeck and put a file in that directory called kieren.html then that file will be visible on the Web at:

 http://iansealy.wdd0708.bbkweb.org/birkbeck/kieren.html

That is, I have appended the directory (birkbeck) and the name of the file (kieren.html) to my base URL (http://iansealy.wdd0708.bbkweb.org/). Each component of the URL is separated with a forward slash.

If I create a subdirectory of birkbeck called college then the URL of that directory will be:

 http://iansealy.wdd0708.bbkweb.org/birkbeck/college/

Index Documents

If I were to put a file called index.html in the birkbeck/college directory then instead of seeing a default directory listing when I went to:

 http://iansealy.wdd0708.bbkweb.org/birkbeck/college/

I would instead see the contents of the index.html document.

The index.html file is just an ordinary file and, in fact, it can be viewed normally. The following two URLs would produce exactly the same page:

 http://iansealy.wdd0708.bbkweb.org/birkbeck/college/
 http://iansealy.wdd0708.bbkweb.org/birkbeck/college/index.html

The DreamHost Web server is configured so that if no index document is present then a directory listing is shown instead. This doesn't have to be the case and Web servers can be configured so that an error message is shown instead. An example of this can be seen at:

 http://www.bized.co.uk/learn/

An index document doesn't have to be called index.html, although this is the most common. Other examples are index.htm, welcome.html, home.html and Default.htm. The latter is commonly used on Windows-based Web servers.

As an example, the Biz/ed Web site doesn't have an index.html in the root directory (the top directory of the site). This means that the following URL does not work:

 http://www.bized.co.uk/index.html

The real index document is actually:

 http://www.bized.co.uk/index.htm

This page is identical to:

 http://www.bized.co.uk/

For another explanation of index documents (as well as more information about the Web in general), have a look at:

 http://daphne.palomar.edu/learnit/web.htm
