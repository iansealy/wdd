What is XML?

EXtensible Markup Language

  • Markup language
  • Focuses on Data
  • Can define your own tags

Example:

<cd>
<artist>Moby</artist>
<title>18</title>
<label>Mute</label>
</cd>

What is XML used for?

  • Sharing data (no presentation)
    • Strict structure - must be well-formed
    • Easily read by different applications
    • Self descriptive - should be possible to write without previous knowledge
    • Plain text, it's not special
    • Platform independent
  • Developing new technologies/languages

XML and HTML

  • XML is for data
  • HTML for displaying data on the web
  • XML is not a replacement for HTML
  • XML describes data; HTML can then be used to format and display the data
  • XHTML is a "rejigging" of HTML 4 as XML
  • You can write HTML5 using the XHTML syntax

XML structure

  • Must begin with
    <?xml version="1.0" encoding="utf-8"?>
  • Must close tags
  • Case sensitive
    • <artist>Moby</Artist> is incorrect
    • <artist>Moby</artist> is correct
  • Tags must be properly nested
    • <foo><bar>Hello world</foo></bar> is incorrect
    • <foo><bar>Hello world</bar></foo> is correct
  • Attributes must be quoted
    • <cd release_date=01/01/02 /> is incorrect
    • <cd release_date="01/01/02" /> is correct
  • White space is preserved
  • Documents must have a single pair of root tags

Document Type Definition (DTD) and Schemas

XML documents are self-descriptive. This is achieved by using DTDs.

  • Define the structure of your XML file
  • Define the permissable elements (tags)

XML example

<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE onmydesk SYSTEM "onmydesk.dtd">
<onmydesk>
	<cd>
		<artist>Moby</artist>
		<title>18</title>
		<label>Mute</label>
	</cd>
	<cd>
		<artist>The Verve</artist>
		<title>Urban Hymns</title>
		<label>Virgin</label>
	</cd>
</onmydesk>

Document Type Definition (DTD) example 


<?xml version="1.0" encoding="utf-8"?>
<!ELEMENT onmydesk	(cd+)>

<!ELEMENT cd	(artist,title,label)>
<!ELEMENT artist	ANY>
<!ELEMENT title		ANY>
<!ELEMENT label		ANY>

AJAX

AJAX stands for Asynchronous Javascript And XML.

Not a new technology but a group of existing technologies:

  • XHTML/HTML5 and CSS
  • JavaScript
  • Document Object Model (DOM)
  • XMLHttpRequest
  • XML (although not always)

Dynamic update of page content without the need to reload the page.

Makes the browser/Web application seem more responsive.

AJAX example: Google suggest

Problems with XML

  • It's verbose
  • It's complicated - easy for machines to read/write... hard for humans

Alternatives:
