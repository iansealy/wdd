Image Maps

An image map is an HTML feature which allows you to specify areas of an image as 'hotspots' that link to other pages or sites. The advantage of an image map is that you can design a navigation toolbar or menu as one image and use an image map to create 'hotspots'. You can also add a map to an existing scanned image.

Image Maps

To create an image map you need to give it a name (using the map tag and the name attribute) and specify the shape of the hotspot, its pixel coordinates and the destination of the link.

Example:

<map name="worldmap">
<area shape="rect" coords="180,130,250,280" href="samerica.html">
<area shape="rect" coords="290,80,410,180" href="africa.html">
<area shape="rect" coords="500,180,580,250" href="australia.htm">
</map>

Image Maps

There are a number of shape options for 'hotspots'. These are:

  • rect
  • circ
  • poly

RECT

For rectangular shapes. The order for coordinates is: top left x and y followed by bottom right x and y.

Example:

shape="rect" coords="180,130,250,280"

CIRC

For circular shapes. The order for coordinates is: centre x and y, radius.

Example:

shape="circ" coords="180,130,100"

POLY

For polygon shapes. The order for coordinates is: x and y for each point/corner.

Example:

shape="poly" coords="100,120,200,220,320,400"

USEMAP

To apply the map you have created to the image add the usemap="#mapname" attribute to the img tag.

Example:

<img src="world.gif" usemap="#worldmap">

Problems with images maps

  • They are invisible to text-only browsers and to graphic browsers with graphics disabled
  • Unlike HTML lists, tables or menus they are not scalable; an image map 700 pixels wide will not be visible on a 640 pixels wide screen
  • The colours and text fonts are fixed, so they cannot be configured by sight-impaired users
  • The graphics files are large
