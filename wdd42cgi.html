Introduction

  • CGI = Common Gateway Interface
  • Method of creating dynamic or interactive Web pages (e.g. database-driven pages)
  • Examples:
    • Biz/ed search engine (URL contains "cgi-bin")
    • Penn World Data graphing tool (URL contains "cgi-bin")
    • 234car contact form (URL does not contain "cgi-bin", but is CGI, as is rest of system)
  • CGI is an interface between the Web server and a program running on the computer hosting the Web server

More Introduction

  • Normally a requested file is sent back to a Web browser directly
  • With CGI the requested file is executed as a program and that program's output is sent back to the Web browser
  • Parameters (e.g. your search term or your email address) are sent to the Web server in a standard way (the Common Gateway)
  • CGI script must be able to get at these parameters
  • CGI scripts can be written in any language that will run on the computer hosting the Web server

Languages

Examples of programming languages that are often used to write CGI scripts:

  • Perl - very popular (I use Perl given the choice)
  • PHP - also very popular
  • VBScript - a Microsoft Windows scripting language (often used with ASP)
  • C and C++ - used when speed is important (because compiled, not interpreted)
  • Python
  • Tcl

Reusing CGI Scripts

Teaching any of these languages is beyond the scope of this course. Many Web developers never write their own scripts. Is easy to reuse scripts that other people have written.

What You Need

  • A cgi-bin on your Web server (an area where files are executed rather than just sent back directly)
  • On the DreamHost server, CGI scripts can be run from any directory below htdocs and scripts just need to have the right file extension (e.g. .pl or .php)
  • Is also possible to use scripts hosted by a third party - CGI Resource Index and ScriptSearch.com

Example

#!/usr/bin/perl

use CGI qw(param);

print "Content-type: text/html\n\n";

print "<html><h1>Hello ", param('username'), "</h1></html>";

Example

  • Is a Perl script
  • Located at http://wdd0506.bbkweb.org/cgi-bin/cgiexample.pl
  • Takes a parameter called "username" and displays the value of that parameter after "Hello"
  • Can pass parameter as part of URL
  • http://wdd0506.bbkweb.org/cgi-bin/cgiexample.pl?username=Ian
  • http://wdd0506.bbkweb.org/cgi-bin/cgiexample.pl?username=Kieren

Forms

  • Using URLs is one way of sending parameters to a CGI script, but most common way is probably to use forms
  • Forms are HTML that allow user to enter information and send it to the Web server
  • All forms are enclosed in <form> elements
  • action attribute of form element points to CGI script
  • Are three main elements within a form - <input>, <select> and <textarea>

input element

<form action="/cgi-bin/cgiexample.pl">
<input type="text" name="username" size="10">
<input type="submit" value="Go">
</form>

More input element

<form action="/cgi-bin/cgiexample.pl">
<input type="radio" name="username" value="Ian">Ian's first name<br>
<input type="radio" name="username" value="Kieren">Kieren's first name<br>
<input type="radio" name="username" value="Ian Sealy">Ian's full name<br>
<input type="radio" name="username" value="Kieren Pitts">Kieren's full name<br>
<input type="submit" value="Go">
</form>

select element

<form action="/cgi-bin/cgiexample.pl">
<select name="username">
<option value="Ian">Ian's first name
<option value="Kieren">Kieren's first name
<option value="Ian Sealy">Ian's full name
<option value="Kieren Pitts">Kieren's full name
</select>
<input type="submit" value="Go">
</form>

textarea element

<form action="/cgi-bin/cgiexample.pl">
<textarea rows="10" cols="40" name="username"></textarea>
<input type="submit" value="Go">
</form>

method attribute

Two ways of accessing a CGI script. Determined by method attribute of form element.

POST:
<form action="/cgi-bin/cgiexample.pl" method="POST">

GET:
<form action="/cgi-bin/cgiexample.pl" method="GET">

Which is used depends on CGI script. Can POST an unlimited amount of data. GET is limited.
