Introduction

Before we look at SSIs, we need to know what happens when you visit a Web site.

  1. All machines on the Internet use an Internet Protocol (IP) address to refer to each other eg: 69.163.195.90
  2. User types a URL in to a browser address bar or follows a link eg:
    http://wdd1011.bbkweb.org/wdd41agenda.html
  3. Browser refers to Domain Name Servers to convert wdd1011.bbkweb.org in to an IP address
  4. The IP address is for the server hosting the web site
    • What's a server? Hardware or software?

Introduction continued

  1. Browser sends a request for /wdd41agenda.html to the server.
  2. Web server receives the request for the page /wdd41agenda.html
  3. HTML file sent back to browser
  4. Browser reads the HTML and sends requests for any ancillary files to the server:
    • CSS
    • Images
    • JavaScript files

Server Side Includes

  • Server Side Includes (SSIs) are directives/instructions within a web page that are parsed by the Web server before the page is returned to the browser.
  • Used to separate presentation from content by storing repetitive HTML fragments in a separate file. Each fragment can then be used in many pages.
  • HTML fragments don't have to represent valid HTML themselves - when combined with the rest of the page the page will be valid HTML.
  • Many Web servers support SSIs.
  • Only covering Apache because it's the most popular Web server and DreamHost uses it.
  • You need a web server, SSIs won't be included if files are viewed offline.

Parsed Files and Syntax

  • Many file types can parsed for SSIs
  • Since parsing a file is more work for the server, files to be parsed are often identified by file extension with .shtml being common (e.g. DreamHost)
  • Basic syntax:
    <!--#element attribute=value -->
  • Example:
    <!--#echo var="DATE_LOCAL" -->
  • <!--#echo var="LAST_MODIFIED" -->
  • <!--#echo var="DATE_GMT" -->

.htaccess files

  • Usually you'd be able to set the format of the time using:
  • <!--#config timefmt="%B %d, %Y" -->
  • Time formatting codes
  • Unfortunately the above formatting doesn't work on Dreamhost so we use a .htaccess file to do the same

.htaccess files - continued

  • .htaccess is just a configuration file that you put in a directory. It allows a custom configuration to be applied to a directory and any sub-directories within it.
  • SSITimeFormat "%B %d, %Y"
  • Files beginning with a . may be hidden in your FTP client so you may need to configure it to display hidden files. You may also have to upload the file as htaccess.txt (for example) and then rename it to .htaccess
  • .htaccess files are very powerful, we'll meet them again later in the course
  • Word of warning - if you get the syntax of a directive within an .htaccess file wrong you'll probably receive a server error. Web servers are very fault intolerant.

Separation of Content and Presentation

Most common use for SSIs

Example file:

<!--#include virtual="ssi/header.html" -->
<p>Content</p>
<!--#include virtual="/ssi/footer.html" -->

Need to create header.html and footer.html in ssi subdirectory

Can use either relative path (ssi/header.html) or absolute path (/ssi/footer.html)

Only need to edit ssi/header.html and ssi/footer.html to change look and feel

File extension is not important, common extensions are .html, .ssi or .txt

Allowing other files to include SSIs

  • Dreamhost's default setup is to only parse .shtml files for SSIs
  • To allow other files to be parsed, create a .htaccess file which contains:
    • AddHandler server-parsed .filenametobeparsed
  • For example, to parse .html, .htm and .ssi files your .htaccess file would contain:
    • AddHandler server-parsed .html
    • AddHandler server-parsed .htm
    • AddHandler server-parsed .ssi
  • Remember, using SSIs means you have to view your pages online.

PHP

  • PHP: Hypertext preprocessor - covered in greater detail later in WDD4.
  • PHP is a scripting language used to create dynamic web pages.
  • User accesses: http://www.example.com/foo.php
    • Web server runs the foo.php program and returns the output
    • Web server doesn't return the contents of foo.php
    • Web browser gets HTML back and is none-the-wiser about how it was generated

PHP includes

  • PHP includes are very similar to SSIs but used in PHP files.
  • Syntax:
    • <?php include("example.php"); ?>
    • Included file can contain HTML or PHP code
    • If PHP code, use <?php and ?> in include file
    • require is almost identical to include but require generates a fatal error if there's an error and the script will stop. include generates a warning and the script still runs.
