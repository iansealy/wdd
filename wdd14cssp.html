CSS Positioning - setting the scene

Most Web sites use HTML tables for layout and the text formatting is achieved through the use of Cascading Style Sheets (CSS).

This approach isn't really 'wrong' (assuming you check your tables linearise properly). However there are good reasons why you might decide against it:

  • Doesn't separate presentation from content
  • Strictly speaking HTML tables should be used for tabular data only
  • Tabular layouts are complicated to maintain
  • Larger filesize

CSS Positioning - setting the scene

Instead of using HTML tables CSS can be used for positioning.

In this session I'm going to introduce you to the basic concepts underlying CSS positioning (CSS-P) and some of the code that makes it possible. This session will not give you definitive answers/solutions to producing particular page layouts. However, it will explain how page elements 'interact' and provide code examples in addition to links to further resources.

The Box Model

Understanding CSS positioning relies on a understanding the Box Model.

Everything on an HTML page fits into a box.

The Box Model - continued

There are two types of box:

  • Block - e.g. tables, div and p tags.
    Form new blocks of content.
  • Inline - e.g. em, strong and span tags.
    Do not form new blocks but lines of content.

All boxes have: Margin, Border, Padding, Content

Div and Span

One of the key building blocks of CSS-P is being able to group elements together using <div> and <span> tags. Think of <div> and <span> tags as containers for other tags.

  • div - block level container (short for division)
  • span - inline container

<div> and <span> have no inherent properties/styles of their own so can be used to apply styles to the content within them.

Browser support

The level of support for CSS positioning in browsers varies.

  • Mozilla 1.6+, Netscape 7, Opera 6+, FireFox etc. are all very good.
  • IE 6 is generally ok but see below.
  • Netscape 4 has very limited support. It will ignore styles rather than attempt to implement them in a lot of cases.
  • IE 5 poses more problems than NN4 due to errors in the way IE 5 interprets the box model.

Browser support

As we've said before you should test your Web site in as many Web browsers as possible before launch.

In addition you should also be aware of browser "Quirks mode". This is a particular problem for IE6 but Microsoft has issued a "hack/fix".

Browser support continued

You can prevent older browsers from attempting to use CSS formatting through importing your stylesheet using this code:

<style type="text/css">
@import url(/style.css);
</style>

Browser support continued

Often you see:

<link type="text/css" rel="stylesheet" href="/nonstandardstyles.css">
<style type="text/css">
@import url(/style.css);
</style>

Older browsers can't use the '@import' function so will only use 'nonstandardstyles.css. Modern browsers will load both stylesheets (nonstandardstyles.css and style.css). If the styles from 'nonstandardstyles.css' are re-defined in 'style.css' then the new styles will be used.

CSS media types

CSS isn't only used to control how a Web page appears on a PC screen. Different stylesheets can be used to make sure your pages print correctly or display differently on the small screen of a mobile phone or a PDA.

Different types of stylesheet are identified using the 'media' attribute. e.g.:

<link rel="stylesheet" type="text/css" media="screen" href="mystyles.css">
<link rel="stylesheet" type="text/css" media="print" href="print.css">
<link rel="stylesheet" type="text/css" media="handheld" href="pda.css">

CSS media types

or if you're importing the CSS:

@import url("mystyles.css");
@import url("print.css") print;
@import url("pda.css") handheld;

Further info: CSS Media types.

Simplifying mark-up
