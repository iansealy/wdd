CSS Positioning - setting the scene

  • Many Web sites use HTML tables for layout and the text formatting is achieved through the use of Cascading Style Sheets (CSS).
  • This approach isn't ideal (you need to check your tables linearise properly) and it's increasingly old fashioned.
  • There are good reasons why tables shouldn't be used for layout:
    • Strictly speaking HTML tables should be used for tabular data only
    • Doesn't separate presentation from content
    • Tabular layouts are complicated to maintain
    • Larger file size for your HTML file

CSS Positioning - setting the scene

  • Instead of using HTML tables, CSS can be used for positioning.
  • In this session we'll look at the basic concepts underlying CSS positioning (CSS-P) and some of the code that makes it possible.
  • This session will not give you definitive answers/solutions to producing particular page layouts. It will provide code examples and explain how page elements 'interact'.
  • CSS-P can be complex, confusing and frustrating to start with. Don't lose heart but keep practising.

Why use CSS positioning?

  • 'Cleaner' Web pages
  • Easier to work with as presentation and content are separate
  • Easier to restructure/re-skin
  • Tables can present significant interoperability problems. For example, browsing using a mobile phone (or other device with a small screen):
    • PDA-ize your site.
  • It's the Web standards way...

The Box Model

Understanding CSS positioning relies on a understanding the Box Model.

Everything on an HTML page fits into a box.

The Box Model - continued

There are two types of box:

  • Block - e.g. h1, p and div tags.
    Form new blocks of content.
  • Inline - e.g. em, strong and span tags.
    Do not form new blocks but lines of content.

All boxes have: Margin, Border, Padding, Content

Div and Span reminder

One of the key building blocks of CSS-P is being able to group elements together using <div> and <span> tags. Think of <div> and <span> tags as containers for other tags.

  • div - block level container (short for division)
  • span - inline container

<div> and <span> have no inherent properties/styles of their own so can be used to apply styles to the content within them.

Most of the examples so far have demonstrated how elements can be arranged using static positioning and floats.

However, there are four positioning schemes (elements can only have one):

  • Static - normal page layout rules
  • Relative - positioned relative to where the box would normally appear
  • Absolute - positioned relative to the parent box/container
  • Fixed - the same as absolute except positioned relative to the browser window

When using relative, absolute or fixed schemes boxes can have four offsets: top, right, bottom and left.

Absolute positioning

In this example Box 2 has an additional attribute of left : 250px and Box 2 has attributes of width : 100px; top : 40px and both boxes have been positioned absolutely (position : absolute;).

Positioning can lead to overlapping of elements.

Browser support

The level of support for CSS positioning in browsers varies.

  • Current versions of Firefox, Safari, Chrome, Mozilla, Opera are all very good.
  • IE6 is poor. IE7 and IE8 are good.
  • If you know your audience still use it... IE5 poses problems due to errors in the way IE5 interprets the box model. IE5 on the Mac is better than IE5 for Windows but is still poor.

Browser support

Test your Web site in as many Web browsers as possible before launch.

In addition you should also be aware of browser "Quirks mode". This is a particular problem for IE6 but Microsoft has issued a "hack/fix".

Browser support continued

Increasingly unnecessary but you can prevent older browsers from attempting to use CSS positioning code through importing your stylesheet using:

<style type="text/css">
@import url(/style.css);
</style>

CSS media types

CSS isn't only used to control how a Web page appears on a PC screen. Different stylesheets can be used to make sure your pages print correctly or display differently on the small screen of a mobile phone or a PDA.

Different types of stylesheet are identified using the 'media' attribute. e.g.:

<link rel="stylesheet" type="text/css" media="screen" href="mystyles.css">
<link rel="stylesheet" type="text/css" media="print" href="print.css">
<link rel="stylesheet" type="text/css" media="handheld" href="pda.css">

CSS media types

or if you're importing the CSS:

@import url("mystyles.css");
@import url("print.css") print;
@import url("pda.css") handheld;

Further info: CSS Media types.

Development approach

  • Suggested approach:
    • Develop in a browser with good support for Web standards:
      • Firefox, Safari, Chrome
      • Don't develop in any version of IE
    • Then test in IE7 and 8 (6 if you know your audience will use it)
    • Edit your CSS and retest
    • Avoid mixing positioning types as much as possible. I tend to work with static (browser default) positioned elements and then use floats. I avoid absolute positioning.
