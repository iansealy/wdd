XHTML and HTML5

Ian Sealy

What Is XHTML?

  • HTML 4 rewritten in XML
  • XHTML 1 was therefore the next version of HTML
    • but now there is also HTML5 (more later...)
  • XHTML 1.0 drafted in 1998 and finalised in 2002 and is still relatively widely used (HTML version statistics)
  • Offers advantages of XML (extensible, machine-readable, etc...), but is backwards compatible with HTML (will work on old browsers)

How Is XHTML Different From HTML?

  • Documents must be well-formed
  • Tags must be in lower case
  • All elements must be closed
  • Attribute values must be quoted
  • Attributes can't be minimised
  • All documents must have a DOCTYPE declaration

Also other trivial differences

Documents Must Be Well-Formed

Elements must be nested correctly.

Current browsers will allow
<strong><em>test</strong></em> (as in test),
but in an XHTML document this would be invalid. It must be
<strong><em>test</em></strong> (as in test).

Tags Must Be In Lower Case

  • Elements and attributes must be in lower case
  • <TD valign="top"> was recommended in HTML 4 (more readable), but in XHTML it must be <td valign="top">
  • Reason why we have recommended this

All Elements Must Be Closed

In HTML 4
<p>1st para<p>2nd para
is valid. In XHTML it must be
<p>1st para</p><p>2nd para</p>

This means that <hr>, <br> and <img> are all invalid.

So <hr> must be <hr></hr> or <hr/>.

Neither of these work well in some really old browsers, so use <hr /> instead.

Attribute Values Must Be Quoted

  • Browsers don't care if you use <tr valign=top> or <tr valign="top">
  • In XHTML it must be <tr valign="top">

Attributes Can't Be Minimised

<hr noshade> is actually an abbreviation of <hr noshade="noshade">. In XHTML must use <hr noshade="noshade">.

Some very old browsers have problems with this (but it's not worth worrying about).

All Documents Must Have A DOCTYPE

An example of an XHTML DOCTYPE:
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/transitional.dtd">

This DTD (document type definition) or DOCTYPE defines the structure of the document (what tags can be used and where they can be used) in a machine-readable format.

DOCTYPEs

  • DOCTYPEs are essential
  • Indicate the version of HTML or XHTML used in a page
  • Web browsers and validators need to know this in order to interpret a page
  • Pages aren't valid without a DOCTYPE
  • Without a DOCTYPE (or with an invalid DOCTYPE) browsers enter quirks mode, which is slower and displays pages differently
  • More Information and List of DOCTYPEs
  • Ideally use a Strict DOCTYPE, but sometimes have to use Transitional (e.g. for iframes)

Why Use XHTML?

  • XHTML DOCTYPEs were, until relatively recently, the most widely used, so you're bound to use XHTML sooner or later
  • XHTML is extensible - new tags can be added
  • XHTML works with older browsers as well as current browsers
  • XHTML is less complex than HTML, so browsers can be simpler and smaller and so will run on a wider range of devices

Converting HTML To XHTML

  • Conversion can be done manually
  • HTML Tidy can automatically convert HTML to XHTML

What's Next?

  • W3C started drafting XHTML 2.0 in 2002
    • Revolutionary approach - how things should be rather than how they are
    • Not backwards compatible
    • Needs to be implemented in an all or nothing fashion
    • e.g. nl tag for navigation lists; hr becomes separator
  • Lots of developer and browser maker resistance

What Instead?

  • WHATWG started work on HTML5 in 2004
  • Founded by browser makers (Apple, Mozilla and Opera), but an open process
    • Evolutionary approach - start from how things are
    • Backwards compatible
    • Can be implemented gradually
  • W3C now also involved
  • Standard still being worked on, but many features are already implemented (e.g. canvas tag)
  • A Brief History of Markup

Differences in HTML5

Some highlights:

  • New elements, e.g. header, footer, nav, audio, video
  • Canvas - use JavaScript to draw graphics on the fly (Example 1 and Example 2)
  • New form controls, e.g. for dates, times, email addresses and URLs
  • Local storage (Example)

Support is not universal (e.g. canvas was only added in IE9). Can use libraries like Modernizr to detect support for new features. (Also are JavaScript libraries to add canvas support to earlier versions of IE.)
